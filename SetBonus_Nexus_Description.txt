A framework that turns matched armour into a reward. Equip several pieces of the same set and gain a constant, lore-flavoured bonus that grows the more of the set you wear — Iron makes you tough, Glass keeps you nimble, Daedric wards off magic, Dark Brotherhood sharpens your sneak attacks. Over [b]140 sets[/b] in all, each themed to its material, culture, or faction (with a nod to ESO-style set design).

Because a single piece can belong to more than one set (a Tamriel Rebuilt Reachman cuirass is both [i]Wormmouth[/i] and [i]Reach[/i]), bonuses [b]stack[/b] — and magnitudes are tuned with that in mind, so committing to a theme is rewarding without breaking the game.

[center][size=5][b]Set Bonus 1.7.1 — Reactive Armour[/b][/size]
[i]Every set now reacts to the moment — and every drawback bites only when it should.[/i]
[color=#888]━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━[/color][/center]

1.7.1 is a content-and-framework update. No new plugin, no save cleaning — drop it in over 1.7.

[size=4][b]✦ Situational drawbacks[/b][/size]
Every set's Weakness is now [b]conditional[/b] instead of a flat, always-on penalty. Each one only bites in its thematic bad moment (at a slightly steeper magnitude to make up for it):
[list]
[*]Metals can't shed a bolt while you're [b]overburdened[/b].
[*]Leathers and furs catch fire near a [b]hearth[/b] (indoors).
[*]Resin and chitin go brittle in the [b]cold dark[/b].
[*]Daedric punishes [b]cowards[/b] who keep their weapon sheathed and unhurt.
[*]Falmer relics still burn in [b]daylight[/b].
[/list]
Set the drawback slider to 0 to skip them entirely, as always.

[size=4][b]✦ A much wider condition vocabulary[/b][/size]
The conditional framework now reads eight new states — identically on [b]both engines[/b], no external tools:
[list]
[*][b]Carry weight[/b] — reward travelling light, or punish a full pack.
[*][b]Faction rank & standing[/b] — guild and Great House gear that grows once you make rank; expulsion has teeth.
[*][b]Bounty[/b] — lawful and holy armour spurns a wanted wearer.
[*][b]Region[/b] — homeland gear that shines where it belongs.
[*][b]Birthsign[/b], [b]werewolf form[/b], and [b]weapon stance[/b] — the stars, the beast-blood, and the drawn blade.
[/list]
These are threaded through the optional [b]Conditional Rebalance[/b]: rank-scaling faction sets, werewolf-blood on Nordic/Orsimer/Companions, Bosmer's light-pack archery, stance-gated assassin strikes, and more — 465 conditional effects across all 140 sets, stacking-verified.

[size=4][b]✦ Full Armor-class sets[/b][/size]
Heavy / Medium / Light Armor now include [b]every[/b] piece of their weight class across the base game and all supported add-ons (Tamriel Rebuilt, OAAB, NOD, AATL — shields included), generated straight from the armour data.

[size=4][b]✦ Optional Flag Companion[/b][/size]
A tiny new opt-in add-on that feeds [b]combat, weather, and movement[/b] to the framework, so mod authors can build sets that react to them. Not needed for the base mod. (MWSE gets all of it; OpenMW gets swimming/movement — its Lua can't read weather or combat — with the hook left open for a dedicated mod.)

[size=4][b]✦ Also fixed[/b][/size]
Enchanted/copied-item matching now checks an item's mesh in addition to its icon. Previously, an unrelated add-on item that happened to reuse a vanilla icon (some NOD pieces share the Indoril helmet icon, for instance) could be wrongly counted toward that set; it no longer can.

[size=5][b]✦ Features[/b][/size]
[list]
[*][b]140 armour sets[/b] across every vanilla material and culture, plus Tribunal & Bloodmoon.
[*][b]Three tiers per set[/b] — bonuses scale up at [b]2[/b], [b]4[/b], and [b]6+[/b] equipped pieces.
[*][b]Conditional bonuses (New in 1.6, expanded in 1.7.1)[/b] — Sets can contain sub-effects that toggle dynamically with your runtime state: dropping below 50% Health, night vs day, indoors vs out, a region of the map, your carry weight, your rank or standing in a faction, your bounty, your birthsign, werewolf form, or having a weapon drawn — singly or combined with AND/OR.
[*][b]Conditional Rebalance (New in 1.7, deepened in 1.7.1, optional)[/b] — don't like the default bonuses? An optional submodule (both engines) replaces every set's effects with a spikier, condition-driven balance. As of 1.7.1 every set's weakness drawback is [i]situational[/i] and the new state vocabulary is threaded throughout. Delete it to go back — details below.
[*][b]Full Armor-class sets (New in 1.7.1)[/b] — the Heavy / Medium / Light Armor sets now include [i]every[/i] piece of their weight class across the base game and all supported add-ons, auto-generated from the armour data.
[*][b]Flag Companion (New in 1.7.1, optional)[/b] — a small opt-in add-on that feeds combat, weather, and movement state to the framework so author-made sets can react to them. Not needed for the base mod; details below.
[*][b]Restrained, flat balance[/b] — modest bonuses that barely grow at the top tier, so they enhance rather than trivialise. Vanilla magic effects only.
[*][b]Thematic trade-offs[/b] — many sets carry a mild Weakness at mid/max (metal → shock, fur/hide → fire, chitin/glass → frost, Altmer → magicka), giving each set an identity and counterplay (it applies to NPCs too).
[*][b]Players and NPCs[/b] — fully-kitted NPCs benefit too (toggleable).
[*][b]Tune it to taste[/b] — two independent sliders scale the [b]benefits[/b] and the [b]weakness drawbacks[/b] separately (25–300% / 0–300%). Want the flavour without the downside? Set drawbacks to 0. Want a bigger power fantasy? Crank benefits up. Changes apply instantly.
[*][b]Enchant-friendly[/b] — set pieces are matched by inventory icon and mesh, as well as by ID, so a player-[i]enchanted[/i] copy of a set item still counts toward the bonus (toggleable). Icon alone isn't used for this — a separate item that merely reuses the same icon (some texture-pack/compilation add-ons do this) won't be pulled in by mistake.
[*][b]Informative tooltips[/b] — hover a set piece to see the set, your current progress, and its effects; the active tier is highlighted. [b]Compact by default (new in 1.7)[/b]: just the active tier's effects — or a greyed preview of what the set gives when it isn't active — with Full (all tiers) and Minimal (bare set names) modes in the options. Built in on MWSE; on OpenMW it works with [url=https://www.nexusmods.com/morrowind/mods/56680]Inventory Extender[/url] installed.
[*][b]No plugin required[/b] — every bonus spell is built at runtime in Lua. Nothing to clutter your load order, no record conflicts.
[*][b]In-game options[/b] — toggle notifications, NPC bonuses, icon matching, debug logging, and the magnitude sliders (MWSE: MCM; OpenMW: Options › Scripts).
[*][b]Mod-aware[/b] — automatically extends to Tamriel Rebuilt, OAAB, NOD, and AATL armour when those are installed.
[*][b]Modder framework (both engines)[/b] — a small interop lets other mods register their own sets: on MWSE via [i]require[/i], on OpenMW via the [i]I.SetBonus[/i] interface or global events.
[/list]

[size=5][b]✦ Balance at a glance[/b][/size]
Constant-effect bonuses; default tiers at [b]2 / 4 / 6+[/b] pieces. All bonuses are [b]Lua-defined and built at runtime (no ESP)[/b], vanilla effects only.
Values are deliberately low and flat (so a full set isn't far above a partial one): elemental/disease resist [b]10 / 15 / 20[/b], magic resist [b]5 / 8 / 12[/b], Reflect & Spell Absorption [b]3 / 5 / 8[/b], Sanctuary/Chameleon [b]5 / 7 / 10[/b], attributes [b]3 / 5 / 8[/b], skills [b]5 / 8 / 12[/b], health [b]8 / 12 / 18[/b], fatigue [b]12 / 20 / 28[/b], attack [b]3 / 5 / 8[/b].
Most sets also carry a mild thematic [b]Weakness[/b] at mid/max ([b]~10 / 20[/b]). The defensive "Shield" effect is replaced with no-VFX equivalents (Resist Normal Weapons / elemental resists) so armoured characters don't all shimmer.
Prefer different numbers? Two in-game sliders scale everything live: a [b]benefit[/b] multiplier and a separate [b]weakness/drawback[/b] multiplier (set it to 0 to drop drawbacks entirely). Magnitudes always stay whole numbers and over-time effects never scale down to nothing.
[i]The full per-set effect breakdown is listed further down this page (and as Set_Bonus_Spell_Reference.md in the download).[/i]

[size=5][b]✦ Optional: Conditional Rebalance submodule (New in 1.7, deepened in 1.7.1)[/b][/size]
Prefer bonuses that react to the fight instead of flat stats? The [b]Conditional Rebalance[/b] is an optional add-on for both engines that re-registers every set's bonuses through the framework's own interop. Each set keeps a lean always-on identity and gains [b]situational spikes[/b] — roughly 1.5–2× stronger than an always-on equivalent, active only while a state holds — themed to the kind of set:
[list]
[*][b]Heavy materials[/b] answer pain: Iron hits harder [i]bloodied[/i] (HP < 50%), Daedric turns terrible on a [i]last stand[/i] (HP < 25%), Dwemer plate reflects in the deep halls it was made for (indoors).
[*][b]Light materials[/b] favour darkness and open sky: Leather sneaks at night, Hide runs outdoors, Glass is untouchable while unscratched.
[*][b]Cultural sets[/b] reward their own: Nordic grants Nords extra Attack, Khajiit gear pounces at night, Redguard kit surges when fatigue runs low.
[*][b]Faction sets[/b] mirror their trade, and [b]scale with your standing (new in 1.7.1)[/b]: guild and Great House gear grows once you make rank, the Temple heals at wayshrines, Thieves Guild peaks on a night-time indoor job — and lawful or holy armour actively [b]spurns a wanted wearer[/b].
[*][b]Class sets[/b] (Light/Medium/Heavy Armor, Cloth) turn purely reactive so they complement — not double — the material sets that share their items.
[/list]
[b]New in 1.7.1 — every drawback is now situational.[/b] Instead of a flat penalty, each set's Weakness bites only in its thematic bad moment (at a slightly steeper magnitude): metals can't shed a bolt under a full pack, leathers and furs catch fire near a hearth, resin and chitin go brittle in the cold dark, Daedric punishes cowards who dodge the fight, Falmer burns in daylight. The condition vocabulary also widened — alongside health/fatigue/magicka, time, sun and location it now reads [b]carry weight, faction rank & standing, bounty, region, birthsign, werewolf form, and weapon stance[/b], singly or in AND/OR blocks. Werewolf-blood wakes on Nordic/Orsimer/Companions, Bosmer favours a light pack, Redguard resonates with the Warrior's stars.
Because items commonly belong to several sets at once (a Chuzei helm is also Bonemold, Dunmer, and Native), overlapping sets are split into [b]lanes[/b] instead of stacking the same stats: the specific set keeps the full-strength identity, the broad cultural umbrellas add only echo-strength flats plus kinship conditionals, and the build tooling rejects any two co-maxable sets that duplicate a full-strength effect or spike the same effect on the same state.
Both magnitude sliders still apply. All 140 sets are covered (73 base + 67 Tamriel Rebuilt), with 465 conditional effects in total; entries for absent add-ons are skipped. Only conditions that evaluate natively on [b]both engines[/b] are used, so MWSE and OpenMW behave identically.
[b]Install:[/b] MWSE — copy [i]MWSE\mods\Static\SetBonusRebalance[/i] in alongside Set Bonus. OpenMW — add [i]OpenMW_SetBonusRebalance[/i] as a data directory and enable [i]setbonus_rebalance.omwscripts[/i] [b]after[/b] setbonus.omwscripts. Uninstall to return to the default balance; no save cleaning needed.
[i]Full design notes and the complete per-set spell list ship as SetBonus_Rebalance_Reference.md in the download.[/i]

[size=5][b]✦ Optional: Flag Companion (New in 1.7.1)[/b][/size]
The framework only ships conditions that read the same on [b]both[/b] engines, so a few states — combat, weather, and movement — are deliberately left out of the shipped sets (OpenMW can't read them where bonuses are evaluated). This tiny opt-in add-on bridges that gap: it detects what each engine can and feeds it to Set Bonus's flag hook, so [b]your own[/b] sets can use [i]combat[/i], [i]swimming[/i], [i]sneaking[/i], [i]moving[/i], and [i]weather[/i] conditions. MWSE provides all five; OpenMW provides swimming/moving (its Lua can't read weather or combat) and leaves the hook open for a dedicated weather/combat mod to supply the rest. Nothing in the base mod or the Rebalance needs it. See SetBonus_Flags_Companion.md.

[size=5][b]✦ Mod compatibility[/b][/size]
Companion modules fold external-mod armour into the matching sets, but only when that mod is actually present:
[list]
[*][b]Tamriel Rebuilt[/b] — hundreds of TR pieces mapped, plus new TR-flavoured sets (Reachmen, Maormer, Ancient Nordic, Mananaut, Dark Brotherhood, Golden Saint, regional guards, and more).
[*][b]OAAB_Data[/b], [b]NOD[/b], [b]AATL[/b] — their armour, bucklers, and artifacts re-linked to the right sets.
[/list]
It edits no armour or NPC records, so it's compatible with essentially everything and load order doesn't matter.

[b]Optional integrations[/b] (auto-detected, ignored if absent):
[list]
[*][b]Consistent Enchanting[/b] (MWSE) — player-enchanted set pieces are matched by their exact base item, so they keep their bonus with no false positives.
[*][b]Inventory Extender[/b] (OpenMW) — provides the tooltip hook OpenMW lacks, enabling the set-info tooltip there.
[/list]

[size=5][b]✦ Installation[/b][/size]
[b]MWSE[/b] — Install with a mod manager, or extract into your Morrowind [b]Data Files[/b] folder so the scripts land in [i]Data Files\MWSE\mods\Static[/i]. No esp to tick.

[b]OpenMW[/b] — Add the [b]OpenMW_SetBonus[/b] folder as a data directory and enable the content file:
[code]data="C:\path\to\OpenMW_SetBonus"
content=setbonus.omwscripts[/code]
Point the data path at the [i]OpenMW_SetBonus[/i] folder itself — the one that directly contains [i]setbonus.omwscripts[/i].

[size=5][b]✦ How it works[/b][/size]
While you wear a set you carry a constant-effect ability whose strength matches how many pieces you have on; swap gear and it updates instantly. The bonus spells are generated in Lua at load, so there are no spell records in any plugin to conflict or leave behind.

[size=5][b]✦ For mod authors (MWSE)[/b][/size]
Register your own sets through a small, stable interop — define bonuses in Lua (built at runtime) or point at your own ESP spells:
[code]local setBonus = require("Static.SetBonus.interop")
setBonus.registerSet{
    name = "My Set",
    items = { "my_helm", "my_cuirass", "my_greaves", "my_boots" },
    bonuses = {
        min = { { effect = "resistFire", magnitude = 5 } },
        mid = { { effect = "resistFire", magnitude = 10 } },
        max = { { effect = "resistFire", magnitude = 20 },
                { effect = "fortifySkill", skill = "longBlade", magnitude = 12 } },
    },
}[/code]

As of 1.6, add declarative condition blocks to gate effects by runtime state:

[code]local setBonus = require("Static.SetBonus.interop")
setBonus.registerSet{
    name = "My Set",
    items = { "my_helm", "my_cuirass", "my_greaves", "my_boots" },
    bonuses = {
        min = { { effect = "resistFire", magnitude = 5 } },
        mid = { { effect = "resistFire", magnitude = 10 } },        
        max = { 
            { effect = "resistFire", magnitude = 20 }, 
            { effect = "fortifySkill", skill = "longBlade", magnitude = 12 } }, -- unconditional base
            -- Add conditions to gate effects by runtime state:
            { effect = "fortifyAttack", magnitude = 15, condition = { kind = "health", op = "<", value = 0.5, fraction = true } },
        },
    },
}[/code]

You can also [b]addItems[/b] to extend existing sets, [b]amendSet[/b] to append items/effects and tweak thresholds, or [b]registerSet[/b] with an existing name to fully replace a set — an empty definition [i]disables[/i] one, so you can override or switch off any set, including this mod's own. Set custom tier thresholds and listen for a [b]Static:SetBonusChanged[/b] event. Full guide is in the download (SetBonus_Interop_Guide.md). The bundled TR/OAAB/NOD/AATL modules are built entirely on this API.

[size=5][b]✦ For mod authors (OpenMW)[/b][/size]
The OpenMW port is a framework too. From a global script, register your sets through the [b]I.SetBonus[/b] interface (or send the matching global events from any script type):
[code]local I = require('openmw.interfaces')
I.SetBonus.registerSet{
    name  = 'My Set',
    items = { 'my_helm', 'my_cuirass', 'my_greaves', 'my_boots' },
    bonuses = {
        min = { { effect = 'resistfire', mag = 5 } },
        mid = { { effect = 'resistfire', mag = 10 } },
        max = { { effect = 'resistfire', mag = 20 },
                { effect = 'fortifyskill', skill = 'longblade', mag = 12 } },
    },
}[/code]

As of 1.6, add declarative condition blocks to gate effects by runtime state:

[code]
I.SetBonus.registerSet{
    name  = 'My Set',
    items = { 'my_helm', 'my_cuirass', 'my_greaves', 'my_boots' },
    bonuses = {
        min = { { effect = 'resistfire', mag = 5 } },
        mid = { { effect = 'resistfire', mag = 10 } },
        max = {
            { effect = 'resistfire', mag = 20 }, 
            { effect = 'fortifyskill', skill = 'longblade', mag = 12 } }, -- unconditional base
            -- Add conditions to gate effects by runtime state:
            { effect = 'fortifyattack', mag = 15, condition = { kind = 'health', op = '<', value = 0.5, 
            fraction = true } },
        },
    },
}
[/code]

Same helpers as MWSE — [b]addItems[/b], [b]amendSet[/b] (append items/effects, tweak thresholds), full-replace or [i]disable[/i] a set via [b]registerSet[/b] (empty = off), plus [b]getSet[/b]/[b]getSets[/b] queries — and, new in 1.7, [b]registerSets(list)[/b] (interface v2) to register many sets in one batch with a single rebuild/recompute. Everything's also available as global events for mods without a global script. Registered sets obey the magnitude sliders automatically and show on Inventory Extender tooltips (condition labels included). Full guide: SetBonus_Interop_OpenMW.md, with a ready-to-use example in the [b]OpenMW_SetBonus_Example[/b] folder.
